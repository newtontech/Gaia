# 05 科学知识形式化

> 本章综述将科学知识进行形式化表示与推理的三个代表性方向：超图知识表示、证明助手、以及语义出版基础设施。Gaia 的定位跨越了这三个方向——它具有超图的结构表达力、证明助手的形式化严谨性以及语义出版的包模型，同时引入了它们都缺少的概率推理语义。

---

## 5.1 Stewart & Buehler — 超图科学知识表示 (2025)

### 5.1.1 系统架构

Stewart 与 Buehler 在 2025 年提出了一个面向生物复合材料领域的科学知识超图系统。其完整流水线为：

1. **语料获取**：收集约 1100 篇生物复合材料（biocomposite）相关论文的全文。
2. **LLM 双通道抽取**：对每篇论文执行两遍 LLM 抽取——第一遍为"语法通道"（grammatical pass），提取论文中的实体、属性和显式关系；第二遍为"语义通道"（semantic pass），提取隐含的功能关系、因果关系和跨领域类比。两个通道的结果合并去重后形成原始知识条目。
3. **超图构建**：使用 HyperNetX 库构建超图。最终规模约为 161,000 个节点与 320,000 条超边（hyperedge）。每条超边可以连接任意多个节点，表达 n-ary 关系。
4. **推理流水线**：在超图上执行 k-shortest path 搜索，找到连接查询起点与终点的多条路径。然后由三个串行 LLM 代理处理：GraphAgent 负责解读图路径语义，Engineer 负责将抽象关系转化为工程方案，Hypothesizer 负责生成新假设。

### 5.1.2 为什么选择超图而非成对知识图谱

该工作的核心论证点在于：传统的成对知识图谱（pairwise KG，即三元组 (subject, predicate, object)）**无法无损表达科学中的 n-ary 关系**。

以化学反应为例：`A + B + 催化剂 → C + D` 涉及五种物质的共同参与关系。若强行用三元组表示，需要对 n 个参与者做团扩展（clique expansion），产生 n(n-1)/2 条两两边。这不仅膨胀了图的规模，更关键的是引入了**虚假的两两关系**——原本只有"五者共同参与"这一个语义，团扩展后却暗示了每两者之间都存在独立关系。在复合材料领域，材料配方（多组分混合）、多步合成路径、多性能联合优化等场景中，这种信息损失尤为严重。

超图通过允许一条超边连接任意多个节点，自然避免了这一问题。Stewart 与 Buehler 的工作实证验证了：在 1100 篇论文的规模上，超图结构确实能保留成对图丢失的组合关系。

### 5.1.3 关键缺失

尽管超图结构选择合理，该系统在推理层面存在重大缺失：

- **无概率语义**：超边没有置信度、强度或任何概率标注。所有知识被平等对待。
- **无 belief propagation**：推理完全依赖 k-shortest path + LLM prompting，没有任何形式化推理算法。
- **无不确定性处理**：系统无法表达"A 以 0.8 的置信度支持 B"。
- **无矛盾处理**：当两条超边蕴含矛盾结论时，系统没有检测或调解机制。
- **无定量评估**：论文没有提供任何基线对比实验、消融实验或定量指标。
- **本质是 GraphRAG**：推理流水线的核心是"图检索 + LLM 生成"，属于 GraphRAG 的超图变体，而非形式化推理系统。

### 5.1.4 与 Gaia 的详细对比

| 维度 | Stewart & Buehler (2025) | Gaia |
|------|--------------------------|------|
| **图结构** | 超图（HyperNetX），节点与超边 | 推理超图（factor graph）：Knowledge 节点 + Operator/Strategy 因子节点 |
| **关系表达** | 超边连接任意多节点，无类型 | 类型化因子（IMPLICATION, CONJUNCTION, SOFT_ENTAILMENT 等），每种有精确势函数 |
| **概率语义** | 无 | 每个 claim 携带 prior 与 belief；因子参数化定义条件概率 |
| **推理算法** | k-shortest path + LLM prompting | Loopy belief propagation（消息传递，收敛到 Bethe 自由能驻点） |
| **不确定性** | 无 | 全面的不确定性传播：前提信念变化自动影响结论信念 |
| **矛盾处理** | 无 | CONTRADICTION 算子显式建模互斥关系；BP 自动平衡矛盾证据 |
| **知识来源** | LLM 自动抽取（grammatical + semantic） | 人类作者通过 DSL 声明（类型化命题 + 推理链） |
| **形式化程度** | 低（非结构化超边标签） | 高（类型化 DSL → 编译为形式化 factor graph） |
| **可验证性** | 不可验证（LLM 黑盒抽取） | 可编译验证（DSL 编译器 + 图结构校验） |
| **规模** | 约 1100 篇论文 → 161K 节点 | 面向包级别细粒度推理，每个包是独立可验证单元 |
| **评估方法** | 定性案例展示 | 提供 belief snapshot 历史与概率追溯机制 |
| **推理类型** | 单一路径检索 | 多种命名推理策略（deduction, induction, abduction, contradiction, retraction） |
| **知识演化** | 静态快照 | 版本化身份 `(knowledge_id, version)`，支持知识更新与回溯 |

### 5.1.5 互相验证与互补

Stewart & Buehler 的工作**验证了 Gaia 的一个核心设计假设**：成对图（传统知识图谱）不足以表示科学知识中的复杂关系。Gaia 从一开始就选择了 factor graph（一种二部超图）作为底层表示，正是基于同样的直觉。

然而 Gaia 在此基础上走得更远：不仅提供超图结构，还通过 factor 势函数赋予每条超边精确的概率语义，并通过 belief propagation 实现自动化推理。Stewart & Buehler 的系统可以看作"有超图结构但无推理语义"的方案，Gaia 则是"有超图结构且有完整推理语义"的方案。

## 5.2 证明助手（Isabelle/HOL, Coq, Lean）

### 5.2.1 确定性逻辑的世界

证明助手（proof assistant / interactive theorem prover）是形式化数学与计算机科学验证的核心工具。主流系统包括：

- **Isabelle/HOL**：基于高阶逻辑（Higher-Order Logic），在数学形式化与程序验证领域有广泛应用。
- **Coq**：基于构造演算（Calculus of Inductive Constructions），以 Curry-Howard 同构为理论基础，证明即程序。
- **Lean**：微软研究院主导开发，集成了强大的元编程框架和自动化策略。Lean 4 及其 Mathlib 数学库（截至 2025 年已包含超过 150,000 个形式化定理）构成了目前最大规模的形式化数学工程。

在证明助手的范式中，命题只有两种状态：**已证明**或**未证明**。证明是一系列从公理出发、通过推导规则逐步到达目标命题的步骤，每一步都经过内核（kernel）机械检验。这是一个**完全确定性**的框架——不存在"某命题以 0.7 的概率为真"这一概念。

### 5.2.2 AlphaProof (2025)

DeepMind 的 AlphaProof 系统代表了 AI 与证明助手结合的最新进展。该系统使用强化学习训练一个代理，在 Lean 环境中搜索证明策略，成功求解了 2024 年国际数学奥林匹克（IMO）的多道难题。AlphaProof 的成功证明了形式化证明搜索可以通过 AI 实现高度自动化。

然而，AlphaProof 的能力边界也清晰揭示了证明助手范式的适用域：它处理的是**有确定答案的数学竞赛题**。对于科学中的开放性问题——例如"高压下氢是否会在室温下超导？"——不存在可以被证明或证伪的形式化陈述，因为问题本身处于不确定的认识状态。

### 5.2.3 根本性哲学分歧

证明助手形式化的是**演绎推理**（deduction）：从已知为真的前提出发，通过逻辑规则推导必然为真的结论。这对应 Jaynes 概率即逻辑框架中概率为 0 或 1 的极端情况。

Gaia 形式化的是**似真推理**（plausible reasoning）：从具有不同置信度的前提出发，通过不确定的推理策略（induction, abduction, deduction）更新结论的置信度。这对应 Jaynes 框架中 (0, 1) 开区间内的一般情况。

在科学实践中，绝大多数命题**不是被证明的，而是具有不同程度的支持**。一个实验结果可能以 0.9 的置信度支持某个假说，另一个实验可能以 0.6 的置信度削弱它。证明助手没有表达这种渐进式知识积累的能力。

### 5.2.4 互补可能性

尽管哲学立场不同，Gaia 与证明助手存在互补空间：

- **演绎核**：对于科学中确实存在确定性推导的部分（数学定理、逻辑推论），可以使用证明助手验证，然后将验证结果作为 prior = 1.0（或极接近 1.0）的 claim 导入 Gaia。
- **似真包络**：围绕演绎核的大量不确定推理（实验支持、归纳概括、类比猜想）由 Gaia 的 BP 推理处理。
- **实际路径**：Gaia 的 deduction 策略（当 p1 接近 1.0, p2 接近 1.0 时的 SOFT_ENTAILMENT）在极限情况下收敛到确定性蕴含，语义上与证明助手的规则应用一致。

## 5.3 Nanopublications 与 ORKG

### 5.3.1 Nanopublications

Nanopublication 是一种基于 RDF 的科学断言微发布格式，由三部分组成：

- **Assertion**：核心科学断言，以 RDF 三元组表达（如 `<DrugX> <treats> <DiseaseY>`）。
- **Provenance**：断言的来源信息（实验方法、数据集、发表论文等）。
- **Publication Info**：nanopublication 自身的元数据（作者、时间戳、数字签名）。

Nanopublication 的设计理念是将科学知识分解为**最小可引用单元**，每个单元都有清晰的来源追溯和不可篡改的身份。截至 2025 年，nanopublication 网络中已有超过数百万条条目，主要集中在生命科学领域。其基础设施支持去中心化发布与检索。

### 5.3.2 Open Research Knowledge Graph (ORKG)

ORKG 是由 TIB（莱布尼茨信息基础设施研究所）主导的开放研究知识图谱项目。其核心功能是从论文中提取结构化的研究贡献描述：

- **研究问题**（Research Problem）
- **方法**（Method）
- **数据集**（Dataset）
- **结果**（Result）
- **对比表**（Comparison）——跨论文同一问题下不同方法的对比

ORKG 通过人工标注与半自动抽取相结合，构建了一个论文级的结构化知识库。用户可以浏览某个研究问题下所有论文的方法与结果对比，但系统本身不执行任何推理。

### 5.3.3 共同的根本缺失

Nanopublication 和 ORKG 都对科学知识进行了**结构化**，但都止步于此：

- **无概率推理**：两者都不为断言附加置信度，也不在断言之间执行任何形式的概率传播。
- **无矛盾检测**：当两个 nanopublication 或两个 ORKG 条目蕴含矛盾结论时，系统既不检测也不调解。
- **静态快照**：知识条目一旦发布即冻结（nanopublication 通过数字签名保证不可变性），不参与动态推理。

### 5.3.4 Gaia 包模型的演进

Gaia 的 Package 模型可以看作 nanopublication 的结构化演进：

| 维度 | Nanopublication | ORKG | Gaia Package |
|------|----------------|------|-------------|
| 粒度 | 单条断言 | 论文级贡献描述 | 模块级（多命题 + 推理链） |
| 语义 | RDF 三元组 | 属性-值对 | 类型化命题 + factor graph |
| 概率 | 无 | 无 | 每个 claim 有 prior 与 belief |
| 推理 | 无 | 无 | BP 自动推理 |
| 来源追溯 | 有（provenance graph） | 有（论文引用） | 有（provenance 列表 + 版本历史） |
| 身份 | URI + 数字签名 | ORKG ID | QID `namespace:package::label` + content_hash |
| 演化 | 不可变（重发为新 nanopub） | 版本化但无推理联动 | 版本化 + belief snapshot 历史 |
| 知识交互 | 引用关系 | 对比表 | 跨包推理（通过 external QID 引用参与 BP） |

关键区别在于：Gaia 的包**不是静态断言，而是动态参与推理的对象**。当新证据（新的包）发布时，它可以通过推理链影响已有包中 claim 的 belief 值。这是 nanopublication 和 ORKG 都无法实现的——它们的知识条目在发布后是惰性的。

## 小结

三个方向各自贡献了科学知识形式化的一个切面：Stewart & Buehler 验证了超图结构的必要性，证明助手展示了形式化验证的力量，nanopublication/ORKG 建立了结构化科学出版的基础设施。Gaia 将这些切面整合为一个统一的 pipeline：**类型化 DSL 声明命题 → 编译为 factor graph 超图 → belief propagation 概率推理**，在结构表达力、形式化严谨性和动态推理能力上同时推进。
