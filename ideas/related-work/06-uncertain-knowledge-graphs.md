# 06 不确定知识图谱

> 本章综述在知识图谱上引入不确定性（概率/置信度）的代表性方法。这些工作与 Gaia 共享"知识应携带概率"的核心直觉，但在技术路径上走向了截然不同的方向——大多数方法通过嵌入学习在连续向量空间中隐式编码不确定性，而 Gaia 通过 factor graph + belief propagation 在显式概率框架中进行推理。

---

## 6.1 UKGE — Embedding Learning with Confidence (Chen et al., 2019)

### 6.1.1 架构

UKGE（Uncertain Knowledge Graph Embedding）是第一个系统性地将置信度分数（confidence score）引入知识图谱嵌入学习的方法。其架构分为三层：

1. **不确定知识图谱输入**：每条三元组 (h, r, t) 附带一个置信度分数 s ∈ (0, 1]，表示该事实为真的概率。来源可以是人工标注、信息抽取系统的置信度输出或统计频率。
2. **嵌入学习**：在 TransE/DistMult 等标准嵌入模型基础上，修改损失函数，使嵌入空间中的评分函数不仅区分正负样本，还要拟合正样本的置信度分数。具体地，对于正样本三元组 (h, r, t, s)，损失函数惩罚模型评分与 s 的偏差。
3. **全局一致性约束（PSL）**：引入 Probabilistic Soft Logic（PSL）规则来施加全局一致性约束。例如，规则 `hasFather(x, y) ∧ hasBrother(y, z) → hasUncle(x, z)` 的 PSL 实例化会约束相关三元组的置信度之间满足 Lukasiewicz t-norm 不等式。PSL 约束通过联合优化融入嵌入学习。

UKGE 提出了两个变体：UKGE(rect) 使用矩形约束保证评分在 [0,1] 内；UKGE(logi) 使用 sigmoid 映射。

### 6.1.2 不确定性类型

UKGE 处理的是**认知不确定性**（epistemic uncertainty）——即由于知识不完整导致的对事实真伪的不确定。置信度分数是点估计，不区分偶然不确定性（aleatoric uncertainty）与认知不确定性。

### 6.1.3 与 Gaia 的对比

| 维度 | UKGE | Gaia |
|------|------|------|
| 概率表示 | 三元组级别的标量置信度 s ∈ (0,1] | Knowledge 节点级别的 prior + belief（区分先验与后验） |
| 推理机制 | 嵌入空间中的向量运算 + PSL 软约束 | Factor graph 上的 loopy BP 消息传递 |
| 推理可解释性 | 低（嵌入空间是黑盒） | 高（每个因子的势函数有明确语义，消息传递路径可追溯） |
| 一致性保证 | PSL 规则的近似满足 | BP 收敛到 Bethe 自由能驻点（全局近似一致） |
| 知识表示 | (h, r, t, s) 四元组 | 类型化命题 + 命名推理策略 |
| 推理类型 | 链接预测（预测缺失三元组） | 信念更新（给定新证据，更新所有相关命题的信念） |
| 矛盾处理 | 无显式机制 | CONTRADICTION 算子 + BP 自动平衡 |

**核心差异**：UKGE 的目标是**学习嵌入以预测缺失事实**，概率是训练信号而非推理对象。Gaia 的目标是**在给定推理结构上传播信念**，概率是推理的核心输入和输出。

## 6.2 BEUrRE — Box Embeddings with Calibrated Probability (Chen et al., 2021)

### 6.2.1 架构

BEUrRE（Box Embedding for Uncertain RElational data）将不确定知识图谱中的实体和关系表示为高维空间中的**盒子**（axis-aligned hyperrectangles），而非点向量或高斯分布。

- **实体盒子**：每个实体 e 对应一个盒子 Box(e) = [l_e, u_e]，其中 l_e 和 u_e 分别是各维度的下界和上界向量。
- **关系盒子**：每个关系 r 对应一个仿射变换，将头实体盒子映射到尾实体盒子附近。
- **概率语义**：三元组 (h, r, t) 的概率由"变换后的头实体盒子"与"尾实体盒子"的**体积交集比**（volume ratio of intersection）给出。这提供了一个几何上直觉的概率计算方式。
- **校准**：BEUrRE 通过 Platt scaling 对模型输出概率进行校准（calibration），使其在统计意义上与真实频率一致。

### 6.2.2 几何与概率的联系

BEUrRE 的关键创新在于将概率语义锚定在几何测度上。盒子的体积越大，表示对该实体可能状态的不确定性越高（类似于概率分布的方差）。两个盒子的交集体积自然给出了条件概率的近似。这种几何方法的优点是全局一致性通过盒子包含关系隐式保证——如果 Box(A) ⊂ Box(B)，则 P(A|B) 自然更高。

### 6.2.3 与 Gaia 的对比

| 维度 | BEUrRE | Gaia |
|------|--------|------|
| 概率来源 | 几何体积交集比（学习得到） | 先验分配（人类指定）+ BP 推理更新 |
| 一致性机制 | 几何包含约束（隐式） | BP 消息传递（显式迭代收敛） |
| 概率校准 | Platt scaling 后校准 | 概率直接由 Jaynes 框架定义，先验由作者基于 MaxEnt 原则指定 |
| 表达力 | 三元组关系 | 任意 n-ary 关系（通过 factor 连接多个 Knowledge 节点） |
| 可解释性 | 低（高维盒子难以可视化） | 高（每个因子有命名推理类型和显式参数） |
| 推理方向 | 单向（query → score） | 双向（BP 在 factor graph 上双向传播消息） |

**核心差异**：BEUrRE 的概率是从数据中学习出来的连续空间测度，适合大规模链接预测。Gaia 的概率是从先验出发经推理传播得到的信念，适合小规模高精度的科学推理场景。两者在"什么是概率"这个哲学问题上就不在同一个框架——BEUrRE 更接近频率主义，Gaia 严格遵循 Jaynes 的贝叶斯主义。

## 6.3 PR-OWL — OWL + 多实体贝叶斯网络 (Costa & Laskey, 2005)

### 6.3.1 架构

PR-OWL（Probabilistic OWL）是将概率推理引入语义 Web 本体（OWL）的最有影响力的尝试。其技术路线是将 OWL 本体与 **MEBN**（Multi-Entity Bayesian Networks，多实体贝叶斯网络）结合：

- **OWL 本体**：定义领域的概念层次、属性和约束（如 `Drug subClassOf ChemicalCompound`）。
- **MEBN 理论**：在 OWL 本体的概念之上定义概率模型。MEBN 使用"模板片段"（MFrag）定义局部贝叶斯网络模板，每个 MFrag 包含上下文节点（Context）、输入节点（Input）和驻留节点（Resident），以及对应的条件概率表（CPT）。
- **实例化**：给定具体实体后，MEBN 模板实例化为一个标准贝叶斯网络（称为 SSBN，Situation-Specific Bayesian Network），然后在其上执行标准贝叶斯推理。

### 6.3.2 语义 Web 的概率化尝试

PR-OWL 的雄心在于为整个语义 Web 技术栈引入概率维度。它定义了一套 OWL 本体（PR-OWL ontology）来描述 MEBN 概念，使得概率模型可以用 RDF/OWL 格式序列化和共享。后续版本 PR-OWL 2 进一步改进了本体结构并提供了工具支持（UnBBayes 系统）。

### 6.3.3 与 Gaia 的对比

| 维度 | PR-OWL | Gaia |
|------|--------|------|
| 本体层 | OWL（描述逻辑） | 无独立本体层；类型系统内建于 DSL（claim/setting/question） |
| 概率模型 | MEBN → SSBN（贝叶斯网络） | Factor graph（比贝叶斯网络更一般） |
| 推理算法 | 标准贝叶斯网络推理（精确或近似） | Loopy BP |
| 模板机制 | MFrag 模板 → 实例化 | 全称 claim with parameters → deduction 实例化 |
| 矛盾处理 | 无（贝叶斯网络是 DAG，不允许环） | CONTRADICTION 算子显式建模 |
| 实用性 | 复杂，实际部署案例极少 | DSL 设计面向可用性，编译器自动处理图构建 |
| 循环推理 | 不支持（SSBN 是 DAG） | 支持（loopy BP 处理含环图） |

**核心差异**：PR-OWL 试图在语义 Web 的重型基础设施（OWL + RDF + SPARQL）上叠加概率推理，结果是系统复杂度极高、实际应用极少。Gaia 选择了更轻量的路径：专用 DSL 直接编译为 factor graph，跳过了本体层的复杂性。此外，PR-OWL 继承了贝叶斯网络的 DAG 限制，无法表达科学中常见的循环支持关系（A 支持 B，B 也支持 A），而 Gaia 通过 loopy BP 自然处理这种情况。

## 6.4 综述视角 (Yan et al., 2024)

### 6.4.1 不确定知识图谱的研究格局

Yan et al. (2024) 的综述将不确定知识图谱的方法划分为几个主要范式：

1. **基于嵌入的方法**（UKGE, BEUrRE, PASSLEAF 等）：将不确定性编码到连续向量空间中，通过几何关系近似概率语义。优点是可扩展性好，缺点是可解释性差且概率校准困难。
2. **基于规则的方法**（PSL, ProbLog, MLN 等）：使用显式逻辑规则定义概率关系，通过编译或采样进行推理。优点是可解释性好，缺点是规则获取困难且可扩展性受限。
3. **基于图神经网络的方法**（R-GCN with uncertainty, GMNN 等）：使用消息传递神经网络在图上传播不确定性。介于前两者之间。
4. **混合方法**：结合嵌入学习和规则约束（如 UKGE 中的 PSL 约束）。

### 6.4.2 Gaia 在格局中的位置

Gaia 不属于上述任何一个现有范式，而是占据了一个独特的位置：

- **与嵌入方法的区别**：Gaia 不学习嵌入，不做链接预测。概率不是从数据中学出来的，而是从先验出发经推理传播。
- **与规则方法的接近**：Gaia 的 factor graph 结构最接近规则方法（特别是 MLN 和 PSL），但有两个关键差异：(1) 推理策略（deduction, induction, abduction）是类型化的，不同类型有不同的势函数语义，而 MLN 中所有子句的权重是同质的；(2) Gaia 使用 BP 而非加权模型计数或投影梯度下降。
- **与 GNN 方法的类比**：Gaia 的 BP 消息传递与 GNN 的消息传递在形式上相似（都是在图上迭代聚合邻居信息），但 Gaia 的消息有精确的概率语义（每条消息是一个概率分布），而 GNN 的消息是学习到的特征向量。
- **独特之处**：在所有方法中，只有 Gaia 同时具备 (1) 类型化的科学命题 DSL，(2) 命名推理策略编译为特定 factor 结构，(3) Jaynes 框架下的 BP 推理。

### 6.4.3 核心洞察

不确定知识图谱领域的一个根本张力是**可扩展性 vs. 可解释性**。嵌入方法在百万级三元组上工作良好但不可解释；规则方法可解释但难以扩展。Gaia 的选择是明确站在可解释性一侧——面向的场景是"数百到数千条精心编写的科学命题"，而非"数百万条自动抽取的事实"。这不是技术限制，而是设计选择：科学推理的核心需求是**推理过程可追溯、可审查**，而非处理海量噪声数据。

---

## 小结

不确定知识图谱领域的核心挑战是如何在知识表示中引入并传播概率。现有方法要么通过嵌入学习隐式编码不确定性（UKGE, BEUrRE），要么通过重型本体框架引入贝叶斯网络（PR-OWL）。Gaia 走了第三条路：**通过类型化 DSL 声明命题和推理关系，编译为 factor graph，用 BP 显式传播信念**。这条路径在可解释性和形式化严谨性上优于嵌入方法，在实用性和表达力上优于 PR-OWL，代价是放弃了大规模链接预测的能力——而这恰恰是 Gaia 有意为之的设计取舍。
