Design Section 4: Python Generation Pipeline & Render Flow

Changes to _graph_json.py, template updates, and end-to-end flow

Python Side Changes (gaia render --target github)

File Change Detail
_graph_json.py Major rewrite
  • Strategy nodes: for each strategy in ir["strategies"], emit a node with type: "strategy", strategy_type, module (inferred from conclusion's module), reason
  • Operator nodes: same treatment for ir["operators"]
  • Edges: split each strategy into N premise edges + 1 conclusion edge, each with role: "premise"|"background"|"conclusion"
  • Modules array: iterate knowledges, group by module, compute counts, derive order from ir["module_order"]
  • Cross-module edges: scan strategies where premise.module ≠ conclusion.module
_github.py Minor update Pass module_order to generate_graph_json(). No other changes — README mermaid generation is independent.
Template pages/ Replace React app
  • New package.json: drop cytoscape*, add elkjs
  • New component files per Section 3 architecture
  • Updated types.ts with new schema interfaces
  • New index.css layout (full-width graph, sliding detail panel)
types.ts Rewrite Add StrategyNode, OperatorNode types. Add ModuleInfo, CrossModuleEdge. Update GraphEdge to use role field.

End-to-End Render Flow

gaia compile pkg/
  → .gaia/ir.json  (knowledges + strategies + operators + module_order)

gaia infer pkg/ --review self_review
  → .gaia/reviews/self_review/beliefs.json
  → .gaia/reviews/self_review/parameterization.json

gaia render pkg/ --target github
  │
  ├─ _copy_react_template()
  │   copies gaia/cli/templates/pages/ → .github-output/docs/
  │   (new ELK-based React app, no cytoscape)
  │
  ├─ generate_graph_json(ir, beliefs, priors, exported)  ← REWRITTEN
  │   │
  │   ├─ knowledge nodes  (id, type, module, belief, prior, ...)
  │   ├─ strategy nodes    (id, type="strategy", strategy_type, module, reason)
  │   ├─ operator nodes    (id, type="operator", operator_type, module)
  │   ├─ edges with roles  (source, target, role="premise|background|conclusion")
  │   ├─ modules[]         (id, order, node_count, strategy_count)
  │   └─ cross_module_edges[] (from_module, to_module, count)
  │   → .github-output/docs/public/data/graph.json
  │
  ├─ beliefs.json, meta.json, sections/  (unchanged)
  │
  └─ README.md with coarse mermaid       (unchanged)

Browser (npm run dev)
  │
  ├─ fetch graph.json
  ├─ useGraphData hook → parse nodes by module
  ├─ Level 1: ModuleOverview
  │   └─ useElkLayout(modules, cross_module_edges)
  │       → ELK computes module box positions
  │       → React SVG renders module boxes + inter-module arrows
  │
  ├─ Click module → Level 2: ModuleSubgraph
  │   ├─ filter nodes/edges for selected module
  │   ├─ add external ref nodes (dashed) for cross-module premises
  │   ├─ useElkLayout(filtered nodes, filtered edges)
  │   │   → ELK layered algorithm: settings top → strategies middle → conclusions bottom
  │   └─ React SVG renders full subgraph with NodeRenderer
  │
  └─ Click node → chain highlight
      ├─ useChainHighlight: BFS upstream from clicked node
      ├─ highlight chain nodes/edges, dim everything else
      ├─ if external ref clicked → navigate to source module, auto-highlight
      └─ DetailPanel slides in with node info + reasoning chain
