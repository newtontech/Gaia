Design Section 3: Data Layer & Component Architecture

graph.json schema changes, new components, and file structure

graph.json Schema Changes

Current — flat edges only

{
  "nodes": [
    { "id": "pkg::label",
      "type": "setting",
      ... }
  ],
  "edges": [
    { "source": "pkg::premise",
      "target": "pkg::conclusion",
      "type": "strategy",
      "strategy_type": "deduction" }
  ]
}

Strategy is edge attribute — no intermediate node, reasoning step invisible

New — strategy as node

{
  "nodes": [
    { "id": "pkg::label",
      "type": "setting",
      "module": "motivation",
      ... },
    { "id": "strat_0",
      "type": "strategy",
      "strategy_type": "deduction",
      "module": "s2_laser",
      "reason": "..." }
  ],
  "edges": [
    { "source": "pkg::premise",
      "target": "strat_0",
      "role": "premise" },
    { "source": "pkg::background",
      "target": "strat_0",
      "role": "background" },
    { "source": "strat_0",
      "target": "pkg::conclusion",
      "role": "conclusion" }
  ]
}

Strategy is a real node — premise→strategy→conclusion chain visible

Migration: Modify generate_graph_json() in _graph_json.py. Add strategy/operator nodes with type: "strategy" | "operator", split each strategy edge into premise→strat + strat→conclusion edges with role field.

New modules field in graph.json

{
  "modules": [
    { "id": "motivation", "order": 0,
      "node_count": 5, "strategy_count": 0 },
    { "id": "s2_laser_assisted_dipole", "order": 1,
      "node_count": 14, "strategy_count": 4 },
    ...
  ],
  "cross_module_edges": [
    { "from_module": "s2_laser_assisted_dipole",
      "to_module": "s3_band_structure",
      "count": 2 }
  ],
  "nodes": [...],
  "edges": [...]
}

Module metadata enables Level 1 overview rendering without scanning all nodes

Component Tree

App
├── Header (package name, breadcrumb nav)
├── GraphView (main area, switches between levels)
│   ├── ModuleOverview        ← Level 1: ELK compound graph
│   │   └── ModuleBox[]       ← clickable module containers
│   └── ModuleSubgraph        ← Level 2: ELK layered DAG
│       ├── KnowledgeNode[]   ← shape by type, color by belief
│       ├── StrategyNode[]    ← hexagon intermediate nodes
│       ├── OperatorNode[]    ← circle logic operators
│       └── ExternalRef[]     ← dashed, clickable cross-module
├── DetailPanel (right sidebar, slides in on node click)
│   ├── NodeInfo              ← title, content, prior→belief
│   ├── ReasoningChain        ← upstream premises list
│   └── AbductionComparison   ← competing hypotheses (kept)
├── ChainHighlighter          ← manages highlight state
│   └── computes upstream BFS on click
└── Minimap                   ← corner viewport indicator

Key Dependencies Change

Package Action Why
cytoscape + cytoscape-dagre Remove Replaced by ELK + React SVG
elkjs Add Hierarchical layout engine (~180KB)
react, react-dom, react-markdown Keep —

No React Flow: direct SVG rendering with React gives us full control over node shapes without the overhead of a graph framework. ELK computes positions, React renders SVG.
