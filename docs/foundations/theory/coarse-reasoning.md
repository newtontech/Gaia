## 1. 动机

在 [reasoning-factor-graph.md](reasoning-factor-graph.md) 中，我们定义了四种逻辑原语算子（蕴含、合取、析取、否定），它们的势函数完全由逻辑真值表确定——一致状态 ψ=1，不一致状态 ψ=0，没有自由参数。

如果作者能够将每一步推理都完全分解为逻辑算子的组合，因子图中就不需要任何额外的参数。这样的因子图是"细因子图"——所有因子都是 p=1 的逻辑约束，系统的全部不确定性都来自节点先验 π。

但完全形式化是不现实的。实际的科学推理中：

- **隐含步骤**：作者跳过了中间推理步骤，直接从前提跳到结论
- **隐含假设**：推理依赖的背景假设没有被显式列为前提
- **近似推理**：推理本身不是严格演绎，而是基于经验、类比或直觉的判断

这些情况下，作者无法（或不值得）把推理过程展开为纯逻辑算子。我们需要一种机制来表示"我认为从 M 可以推出 C，但推理过程没有完全形式化"。

粗推理算子就是这种机制。它是因子图中**唯一**携带自由参数 p < 1 的因子类型。

## 2. 粗推理算子定义

**定义：** 粗推理算子是一个二元因子，连接输入命题 M（前提）和输出命题 C（结论），携带参数 p ∈ (0,1]。

**势函数：**

| M | C | ψ |
|---|---|---|
| 1 | 1 | 1 |
| 1 | 0 | f(p) |
| 0 | 1 | 1 |
| 0 | 0 | 1 |

### 四种状态的逐一分析

**(1,1) — 前提成立，结论成立：** 推理生效，这是最受偏好的状态。ψ = 1。

**(1,0) — 前提成立，结论不成立：** 推理失败。如果作者声称"M 支持 C"，但 M 为真而 C 为假，说明推理有问题。p 越大（形式化程度越高），这种状态越不被允许。ψ = f(p)，其中 f(1) = 0——当 p = 1 时完全禁止此状态，退化为逻辑蕴含。

**(0,1) — 前提不成立，结论为真：** 结论可能因为其他推理路径而为真，本因子对此没有意见。ψ = 1。

**(0,0) — 前提不成立，结论不成立：** 同上，前提不成立时因子对结论没有意见。ψ = 1。

### ψ(0,\*) = 1 的选择理由

当前提 M 不成立时，势函数对结论 C 的两种取值给出相同的偏好（ψ = 1）。这是**最小假设**选择：前提不成立时，本因子不对结论提供任何信息。

这个选择的含义是：粗推理算子只在前提成立时"发言"。如果 M 为假，无论 C 是真是假，本因子都不施加任何约束。这避免了在前提不成立时引入额外的建模假设。

### f(p) 的约束条件

f(p) 必须满足以下约束：

1. **f(1) = 0** — p = 1 时退化为蕴含，完全禁止 (M=1, C=0) 状态
2. **f 关于 p 单调递减** — p 越大（形式化程度越高），(M=1, C=0) 状态越受抑制
3. **0 ≤ f(p) ≤ 1** — 势函数取值的基本约束

f(p) 的具体函数形式（如 1-p 或 (1-p)/p）不属于 theory 层的定义范围，留给 [../bp/potentials.md](../bp/potentials.md) 根据计算需要确定。

## 3. 与 Jaynes 弱三段论的一致性

[plausible-reasoning.md](plausible-reasoning.md) 中定义了弱三段论的四条准则 C1-C4。粗推理算子的势函数加上 BP 消息传递，自然产生满足 C1-C4 的行为：

**C1（C 为假 → M 更不可信）✓：** 反向消息方向。当 C 的 belief 较低时，因子向 M 传递的消息会压低 M 的 belief。直觉：如果结论不成立，那么支持该结论的前提的可信度应该下降。

**C2（M 为真 → C 更可信）✓：** 前向消息方向。当 M 的 belief 较高时，因子向 C 传递的消息会提升 C 的 belief。直觉：如果前提成立，那么结论的可信度应该上升。

**C3（C 为真 → M 更可信）✓：** 反向消息方向。当 C 的 belief 较高时，因子对 M 的抑制减弱，M 的 belief 相对提升。直觉：如果结论成立，那么支持该结论的前提的可信度应该上升（弱溯因）。

**C4（M 为假 → C 变弱）✓：** 前向支持撤回。当 M 的 belief 较低时，本因子对 C 的前向支持减弱，C 回落到其先验水平。直觉：如果前提不成立，结论失去这条推理路径的支持。

**重要说明：** 粗推理算子是满足 C1-C4 的**一种**模型——它代表的是最小假设的选择（ψ(0,\*) = 1）。C1-C4 是必要条件，不是充分条件；可能存在其他满足 C1-C4 的势函数设计。我们选择这个设计是因为它的最小假设性质。

## 4. p 的含义

p 的含义是**推理形式化的完整程度**，不是"逻辑关系有噪声"。

- **p < 1** 意味着推理过程中存在未被显式表达的步骤或假设。推理的逻辑骨架是合理的，但作者没有（或无法）将其完全分解为逻辑算子。
- **p → 1** 时，粗推理算子收敛为蕴含算子。f(1) = 0 完全禁止了 (M=1, C=0) 状态，势函数退化为逻辑蕴含的真值表。
- **p 是唯一的因子级自由参数。** 在整个因子图体系中，逻辑算子（蕴含、合取、析取、否定）不携带任何自由参数，它们的势函数由真值表唯一确定。粗推理算子的参数 p 是因子层面引入的唯一自由度。系统的另一个自由度是节点先验 π，但 π 附着在变量节点上，不在因子上。

## 5. 分解模式

粗推理算子与四种逻辑原语（蕴含、合取、析取、否定）的组合可以表达所有不确定关系。以下是基本的分解模式：

### 单前提推理

最简单的情况：一个前提 M 支持一个结论 C，推理未完全形式化。

```
M ──[粗推理 p<1]──→ C
```

直接使用粗推理算子 M→C (p<1)。

### 多前提联合推理

多个前提共同支持一个结论。先用逻辑合取将前提聚合为一个中间命题，再用粗推理连接到结论。

```
A₁ ──┐
A₂ ──┤──[合取 p=1]──→ M ──[粗推理 p<1]──→ C
...  │
Aₙ ──┘
```

合取因子 A₁∧...∧Aₙ→M (p=1) 是纯逻辑约束，不携带自由参数。不确定性完全由粗推理因子 M→C (p<1) 承载。

### 粗等价

两个命题之间存在不完全形式化的双向支持关系。用两个粗推理因子表示：

```
A ──[粗推理 p₁<1]──→ B
B ──[粗推理 p₂<1]──→ A
```

p₁ 和 p₂ 可以不同，反映两个方向的形式化程度可能不同。当 p₁ = p₂ = 1 时退化为逻辑等价。

### 粗矛盾

命题 B 与命题 A 之间存在不完全形式化的矛盾关系。用否定算子引入 A 的否定，再用粗推理连接：

```
A ──[否定 p=1]──→ ¬A
B ──[粗推理 p<1]──→ ¬A
```

否定因子 (A, ¬A) (p=1) 是纯逻辑约束，保证 A 和 ¬A 真值互补。粗推理 B→¬A (p<1) 表示"B 为真在一定程度上支持 A 为假"。

## 6. 粗因子图与细因子图

### 细因子图

所有因子都是逻辑算子（p=1），势函数完全由真值表确定。系统的全部不确定性来自节点先验 π 和图的拓扑结构。这是完全形式化的理想状态。

### 粗因子图

包含粗推理算子（p<1）的因子图。粗推理算子是对未展开子图的近似——它代表了一组尚未被分解为逻辑算子的推理步骤。

### 细化（Refinement）

将粗推理算子展开为逻辑算子组合加上中间命题的过程：

1. 识别粗推理算子中隐含的中间步骤
2. 为每个中间步骤创建新的命题节点
3. 用逻辑算子（蕴含、合取等）连接这些命题
4. 新引入的子图替代原来的粗推理算子

细化后，原来的单个粗推理因子 (p<1) 被替换为若干逻辑因子 (p=1) 加上新的命题节点。不确定性从因子参数转移到了新命题的先验 π 中。

### 实践工作流

**先粗后细** 是推荐的知识构建方式：

1. **构建粗因子图**：捕捉推理的骨架——哪些命题存在，它们之间有什么支持/反对关系。每条推理关系用粗推理算子表示，作者给出 p 值作为形式化程度的估计。
2. **逐步细化**：选择关键的粗推理算子，将其展开为更精细的逻辑结构。每次细化都是局部操作——只替换一个粗推理算子，不影响图的其余部分。
3. **持续改进**：细化不需要一步到位。粗因子图本身已经可以运行 BP 推理，产生有意义的 belief 值。细化是提高推理精度的持续改进过程。

粗因子图不是"不完整的"因子图——它是对知识现状的忠实表示。当推理过程确实没有被完全分解时，用 p < 1 的粗推理算子来表示比强行使用 p = 1 的逻辑算子更诚实、更有用。
