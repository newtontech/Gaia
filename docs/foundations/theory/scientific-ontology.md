# 科学本体论

> **Status:** Target design — foundation baseline
>
> 本文档定义 Gaia 的科学对象模型，供后续基础文档复用。

## 1. 目的

本文档在语言和 BP 细节分化之前，回答四个基本问题：

1. Gaia 中存在哪些科学对象？
2. 哪些对象具有真值、可以参与 BP？
3. 哪些操作是确定性蕴含，哪些是概率性支持？
4. 推理算子如何从暂定阶段成熟为永久结构？

本文档对本体论和建模边界具有规范性。

本文档**不**定义：

- 编写语言的具体语法
- Graph IR 的字段布局
- 因子势函数的公式
- 服务器 API 合约

## 2. 第一原则

Gaia 不是孤立地形式化抽象逻辑。它形式化的是**具有证据溯源、适用条件、不确定性和可修正性的科学断言**。

核心边界：

- **只有封闭的、具有真值的科学断言参与 BP**
- **开放模板、发现工作流和研究任务不直接参与 BP**
- **Claim 的认识论身份由内容和连接它的推理算子决定，而非由 type 编码**

## 3. 科学推理的独特之处

与形式演绎逻辑相比，科学推理有五个额外层次：

1. **世界接口**：前提来自观测、测量、实验和文献——而非来自公理。
2. **不确定性**：我们很少证明一个命题为真；我们评估的是在当前证据下它的可信程度。
3. **适用条件**：科学定律几乎总是附带体系限制、理想化条件和背景假设。
4. **可废止性（defeasibility）**：新证据可以削弱旧结论。矛盾不是系统故障，而是知识更新信号。
5. **开放式发现**：溯因推理、归纳推理和隐含前提的发现都超出纯演绎逻辑的范畴。

## 4. 科学知识推理的普适结构

所有科学推理都遵循同一个基本结构：**前提 → 推理 → 结论**。

```
前提 (Premises)          推理算子 (Reasoning)         结论 (Conclusion)
┌──────────────┐         ┌──────────────┐          ┌──────────────┐
│ Claim A      │────────>│              │─────────>│              │
│ Claim B      │────────>│   Factor     │          │  Claim C     │
│ ...          │────────>│              │          │              │
└──────────────┘         └──────────────┘          └──────────────┘
```

这三个角色是不可再分的：

- **前提（Premises）**：已有的知识断言，作为推理的输入。可以是一个或多个。
- **推理算子（Reasoning Operator）**：连接前提与结论的推理关系。它决定了前提如何支持结论——是确定性蕴含、概率性归纳、还是溯因解释。每个算子携带一个强度（probability），表达该推理步骤的可信程度。
- **结论（Conclusion）**：推理产出的新断言。

这个三元组是 Gaia 中所有知识结构的原子单元。一篇论文、一个实验、一条推理链，最终都分解为这样的三元组的组合。因子图就是这些三元组的集合——前提和结论是变量节点，推理算子是因子节点。

几个推论：

1. **前提和结论是同一种对象**——都是 Claim。一个推理的结论可以成为另一个推理的前提，形成推理链。
2. **推理算子不是装饰性标签**——它携带概率语义，直接参与 BP 信念传播。
3. **没有孤立的知识**——一个没有任何推理算子连接的 Claim 只保留其先验信念，无法被更新。这不是错误状态，但意味着该断言尚未被纳入推理网络。

## 5. 知识对象类别

### 5.1 Claim（断言）

Claim 是一个封闭的、具有真值的科学断言。它是 Gaia 中唯一的 BP 承载对象类别。

示例：

- "在月球真空中，羽毛和锤子以相同速率下落。"
- "该样本在 90 K 以下表现出超导性。"
- "对于真空中靠近地球的刚体，加速度与质量无关。"

**Type 是程序化操作的标识，不是认识论分类。** Claim 不区分假说、定律、预测等——这些认识论身份由两件事决定：

1. **内容**：生产过程（观测、推导、归纳等）体现在断言文本中
2. **连接它的推理算子**：entailment 的结论 = 抽象/特化，induction 的结论 = 泛化候选

每个 Raw Claim 有单一的 author 和单一的生产过程。

#### Claim 的结构特化

虽然所有 Claim 共享相同的 type，但不同来源的 Claim 可以携带不同的结构化元数据：

- **Observation（观测）**：携带溯源元数据——来源、方法、实验条件、观测时间等。其主要权威性来自观测而非推导。
- **Measurement（测量）**：Observation 的特化，额外携带定量结构——量值、单位、校准信息和不确定度。例如："转变温度为 92 ± 1 K。"
- **RegimeAssumption（体系假设）**：背景条件、理想化假设或适用性约束（如"真空"、"非相对论体系"、"近地近似"）。在 Jaynes 框架中，假设只是当前被放在条件概率 P(A|I) 右侧的命题——任何假设都可以被提升为显式变量接受概率评估。其"假设"角色由它在因子图中的位置（作为 factor 的 premise）决定，而非由 type 编码。

这些特化不构成独立的 type——它们是 Claim 元数据 schema 的变体，用于约束和验证 Claim 的结构完整性。

### 5.2 Template（模板）

Template 是一个开放的命题模式或谓词形式。

示例：

- `falls_at_rate(x, medium)`
- `critical_temperature(material, pressure)`
- `P(x) -> Q(x)`

Template 类似于谓词或开放公式。它**不是**封闭命题，不直接携带真值信念。

Template **不**进入 BP。

### 5.3 Setting（背景设定）

Setting 是研究的背景信息或动机性叙述，为理解一项研究提供上下文。

示例：

- 某个领域的研究现状
- 一组实验的动机和出发点
- 已知的未解决挑战

Setting 不是命题，没有真值，不参与 BP。它的作用是为 Claim 和推理结构提供可读的叙事性上下文。

### 5.4 Question（问题）

Question 是一个探究制品，不是具有真值的命题。

示例：

- 未解决的科学问题
- 后续调查目标

Question 不直接进入 BP。

## 6. 推理算子

### 6.1 算子生命周期

推理算子（连接知识节点的因子）不是一次性创建的。它们经历三个阶段：

| 阶段 | 说明 |
|------|------|
| **infer** | 作者写入的推理链接。这是所有推理算子的初始占位符类型，表示"此处存在某种推理关系"，但具体语义尚未经过审查确认。 |
| **candidate** | review/research agent 分析后提议的具体类型。已识别出语义，但尚未经过充分验证。 |
| **permanent** | 经过 research/review agent 研究和判断后确认。正式成为图结构的一部分，具有明确的 BP 规则。 |

**特殊情况**：instantiation（从 Template 到具体 Claim 的实例化）是 entailment 的特例。由于其确定性性质，它可以跳过 review 直接升格为 permanent。

### 6.2 算子类型

以下类型适用于 candidate 和 permanent 阶段：

#### entailment（蕴含）

封闭断言之间的确定性保真关系。如果 A 蕴含 B，则：

- A 支持 B
- ¬A **不**一般意味着 ¬B

entailment 覆盖以下子场景：

- **抽象**：多个具体 Claim 都蕴含一个更弱的公共结论
- **instantiation（实例化）**：从全称定律或模式断言推导出一个具体的封闭断言。这是 entailment 的特例，可跳过 review 直接升格为 permanent

#### induction（归纳）

从具体案例到更广假说或候选定律的概率性支持。成员断言支持结论，但**不**单独蕴含它。

induction 不保真，必须与 entailment 区分开来。

#### abduction（溯因）

从观测到最佳解释的推理。提出一个解释性候选来说明为什么观测到了某些证据。

abduction 是概率性的，且方向与 entailment 相反（从结果推向原因）。

#### equivalent（等价）

相关断言应在真值上一致。

#### contradict（矛盾）

相关断言不应同时为真。

## 7. 图的不可变性与可废止性

Gaia 的知识图**不支持删除操作**。一旦知识节点或推理链接被写入图中，它们不会被物理移除。这种设计保证了图的审计完整性：所有曾经存在的知识和推理关系都有迹可循。

科学信念随新证据而变化。这通过三种机制形式化：

- **通过 BP 的信念更新**：当新因子（推理链接）被添加到图中时，BP 重新计算所有信念。一个先前高信念的断言可能被新的矛盾证据降低。
- **矛盾（contradiction）**：明确声明两个断言不应同时为真的关系。BP 自动削弱支持较少的断言。
- **撤回（retraction）**：由原作者发起，将该知识节点关联的所有 factor 的 probability 设为 0。该知识节点变成孤岛——不再接收来自任何 factor 的消息，belief 自然回到 prior。撤回不是擦除历史，而是通过 BP 的自然机制使某个断言失去证据支持。

可废止性不是缺陷或特殊情况——它是区分科学推理与演绎证明的核心特征。在 Gaia 中，非单调性（新前提可以降低旧结论的信念）被内建于推理引擎中。

## 8. 审查/策展边界

本体论不将审查制品混入知识内容。

- 策展发现的潜在矛盾**尚不是**已接受图中的矛盾断言
- 佐证/独立证据发现**不是** BP 因子
- infer 阶段的推理链接**尚不是**具有明确 BP 语义的 permanent 算子——但它仍然参与 BP。infer 使用通用的默认 potential 进行信念传播，只是其具体的推理类型尚未经过审查确认。审查的作用是将 infer 细化为具有精确 BP 规则的 permanent 算子，而非决定它是否参与 BP。

这些制品在被相应工作流接受之前，仍然是审查/策展制品。但"未审查"不等于"不参与推理"。

## 9. 对后续文档的约束

后续文档应遵循以下规则：

1. plausible-reasoning.md 应解释科学推理为何需要这一本体论
2. 编写语言规范应仅暴露面向作者的子集
3. Graph IR 应定义已接受本体论对象的结构表示
4. 推理理论应仅在承载 BP 的子集上定义 BP 算子合约
5. 审查/策展文档应定义非 BP 制品如何被提议、审计和可能被接受
6. 推理算子的生命周期（infer → candidate → permanent）应在审查流程文档中详细描述
