# 科学本体论

> **Status:** Target design — foundation baseline
>
> 本文档定义 Gaia 的科学对象模型和推理算子分类，供后续基础文档复用。
> 本文档回答"有什么"和"什么能参与推理"的问题，不回答"推理如何计算"的问题。
> 关于推理超图的数学结构和 BP 算法，参见 [belief-propagation.md](belief-propagation.md)。

## 1. 目的

本文档在语言和 BP 细节分化之前，回答四个基本问题：

1. Gaia 中存在哪些科学对象？
2. 哪些对象具有真值、可以参与 BP？
3. 哪些操作是确定性蕴含，哪些是概率性支持？
4. 推理算子如何从暂定阶段成熟为永久结构？

本文档对本体论和建模边界具有规范性。

本文档**不**定义：

- BP 算法细节、消息传递规则或势函数设计（参见 [belief-propagation.md](belief-propagation.md)）
- 编写语言的具体语法
- Graph IR 的字段布局
- 服务器 API 合约

## 2. 第一原则

Gaia 不是孤立地形式化抽象逻辑。它形式化的是**具有证据溯源、适用条件、不确定性和可修正性的科学断言**。

核心边界：

- **只有 Claim（封闭的、具有真值的科学断言）参与 BP**
- **Setting、Question、Template 不直接参与 BP，但可通过 Template 实例化为 Claim**
- **Claim 的认识论身份由内容和连接它的推理算子决定，而非由 type 编码**

## 3. 科学推理的独特之处

与形式演绎逻辑相比，科学推理有五个额外层次：

1. **世界接口**：前提来自观测、测量、实验和文献——而非来自公理。
2. **不确定性**：我们很少证明一个命题为真；我们评估的是在当前证据下它的可信程度。
3. **适用条件**：科学定律几乎总是附带体系限制、理想化条件和背景假设。
4. **可废止性（defeasibility）**：新证据可以削弱旧结论。矛盾不是系统故障，而是知识更新信号。
5. **开放式发现**：溯因推理、归纳推理和隐含前提的发现都超出纯演绎逻辑的范畴。

## 4. 科学知识推理的普适结构

所有科学推理都遵循同一个基本结构：**前提 → 推理 → 结论**。

```
前提 (Premises)          推理算子 (Reasoning)         结论 (Conclusion)
┌──────────────┐         ┌──────────────┐          ┌──────────────┐
│ Claim A      │────────>│              │─────────>│              │
│ Setting X    │────────>│   Factor     │          │  Claim C     │
│ Question Q   │────────>│              │          │              │
└──────────────┘         └──────────────┘          └──────────────┘
```

这三个角色是不可再分的：

- **前提（Premises）**：已有的知识对象，作为推理的输入。可以是一个或多个。前提可以是 Claim、Setting（为推理提供背景上下文）或 Question（驱动推理的探究方向）。
- **推理算子（Reasoning Operator）**：连接前提与结论的推理关系。它决定了前提如何支持结论——是确定性蕴含、概率性归纳、还是溯因解释。每个算子携带一个强度（probability），表达该推理步骤的可信程度。
- **结论（Conclusion）**：推理产出的新断言。

这个三元组是 Gaia 中所有知识结构的原子单元。一篇论文、一个实验、一条推理链，最终都分解为这样的三元组的组合。这些三元组的集合构成推理超图（因子图）——前提和结论是变量节点，推理算子是因子节点。因子图的数学结构和计算语义参见 [belief-propagation.md](belief-propagation.md)。

几个推论：

1. **前提和结论是知识对象**——前提可以是 Claim、Setting 或 Question，结论是 Claim。一个推理的结论可以成为另一个推理的前提，形成推理链。
2. **推理算子不是装饰性标签**——它携带概率语义，直接参与 BP 信念传播。
3. **没有孤立的知识**——一个没有任何推理算子连接的 Claim 只保留其先验信念，无法被更新。这不是错误状态，但意味着该断言尚未被纳入推理网络。

## 5. 知识对象类别

Gaia 中有四种知识对象。**Claim 是唯一默认携带 probability 并参与 BP 的类型。** 其他类型不直接参与 BP，但可以通过 Template 实例化为 Claim 从而获得概率语义。

### 5.1 Claim（断言）

Claim 是一个封闭的、具有真值的科学断言。它默认携带 probability（prior + belief），是 Gaia 中唯一的 BP 承载对象类别。

示例：

- "在月球真空中，羽毛和锤子以相同速率下落。"
- "该样本在 90 K 以下表现出超导性。"
- "对于真空中靠近地球的刚体，加速度与质量无关。"

Claim 可以携带描述其产生方式的结构化元数据。概念性的例子包括：

- 观测条件（来源、方法、实验环境）
- 定量测量值与不确定度（量值、单位、校准信息）
- 计算方法与参数（软件、算法、收敛条件）

这些仅为概念性示例，不构成封闭的分类体系。具体的元数据 schema 由下层文档定义。

### 5.2 Setting（背景设定）

Setting 是研究的背景信息或动机性叙述，为理解一项研究提供上下文。

示例：

- 某个领域的研究现状
- 一组实验的动机和出发点
- 已知的未解决挑战
- 某种近似方法或理论框架

Setting 本身不携带 probability，不参与 BP。它可以直接作为推理的前提参与图结构（提供背景上下文），但不参与信念传播。

### 5.3 Question（问题）

Question 是一个探究制品，表达待研究的方向或未解决的科学问题。

示例：

- 未解决的科学问题
- 后续调查目标

Question 本身不携带 probability，不参与 BP。它可以直接作为推理的前提参与图结构（驱动探究方向），但不参与信念传播。

### 5.4 Template（模板）

Template 是一个开放的命题模式，含有自由变量。

示例：

- `falls_at_rate(x, medium)`
- `{method} can be applied in this {context}`
- `{Q} is an important question`

Template **不**直接进入 BP——它没有封闭的真值。

Template 的核心作用是**桥梁**：它可以将 Setting 或 Question 包装成 Claim，使其获得概率语义并参与 BP。例如：

- Setting "mean field approximation" + Template `"{method} can be applied in this {context}"` → Claim "Mean field approximation can be applied in this condensed matter system"
- Question "室温超导是否可能？" + Template `"{Q} is an important question"` → Claim "室温超导是否可能是一个重要问题"

## 6. 算子

连接知识节点的因子分为三大类：**推理算子**、**计算算子**和**证明算子**。

### 6.1 初始算子类型

作者写入知识图时，使用三种初始算子：

| 初始类型 | 说明 |
|---------|------|
| **infer** | 推理链接。表示"此处存在某种推理关系"（演绎、归纳、溯因等），但具体语义尚未经过审查确认。这是所有推理算子的默认状态。 |
| **toolcall** | 计算链接。表示结论是由计算过程（工具调用、模拟、数值求解等）产生的。前提是计算的输入，结论是计算的输出。 |
| **proof** | 形式化证明链接。表示结论通过严格的形式逻辑从前提推导而来（定理证明、形式验证等）。与 infer 不同，proof 不是概率性的——如果证明有效，结论从前提确定性地成立。 |

三者的区别：infer 的结论来自人类或 agent 的推理判断（概率性），toolcall 的结论来自计算过程（确定性），proof 的结论来自形式化推导（确定性且可验证）。

### 6.2 推理算子的生命周期

infer 类型的算子经历三个阶段：

| 阶段 | 说明 |
|------|------|
| **infer** | 粗粒化的推理过程。 |
| **candidate** | review/research agent 分析后提议的具体推理类型。已识别出语义，但尚未经过充分验证。 |
| **permanent** | 经过 research/review agent 研究和判断后确认。正式成为图结构的一部分，具有明确的 BP 规则。 |

**特殊情况**：instantiation（从 Template 到具体 Claim 的实例化）是 entailment 的特例。由于其确定性性质，它可以跳过 review 直接升格为 permanent。

toolcall 和 proof 不经历这个生命周期——它们的语义在创建时就是明确的（计算过程和形式证明本身定义了前提到结论的关系）。

### 6.3 推理算子类型

以下类型适用于 candidate 和 permanent 阶段：

#### entailment（蕴含）

封闭断言之间的确定性保真关系。如果 A 蕴含 B，则：

- A 支持 B
- ¬A **不**一般意味着 ¬B

entailment 覆盖以下子场景：

- **抽象**：多个具体 Claim 都蕴含一个更弱的公共结论
- **instantiation（实例化）**：从全称定律或模式断言推导出一个具体的封闭断言。这是 entailment 的特例，可跳过 review 直接升格为 permanent

#### induction（归纳）

从具体案例到更广假说或候选定律的概率性支持。成员断言支持结论，但**不**单独蕴含它。

induction 不保真，必须与 entailment 区分开来。

#### abduction（溯因）

从观测到最佳解释的推理。提出一个解释性候选来说明为什么观测到了某些证据。

abduction 是概率性的，且方向与 entailment 相反（从结果推向原因）。

#### equivalent（等价）

相关断言应在真值上一致。

#### contradict（矛盾）

相关断言不应同时为真。

## 7. 图的不可变性与可废止性

Gaia 的知识图**不支持删除操作**。一旦知识节点或推理链接被写入图中，它们不会被物理移除。这种设计保证了图的审计完整性：所有曾经存在的知识和推理关系都有迹可循。

科学信念随新证据而变化。这通过三种机制形式化：

- **信念更新**：当新的推理链接被添加到图中时，推理引擎重新计算所有信念。一个先前高信念的断言可能被新的矛盾证据降低。
- **矛盾（contradiction）**：明确声明两个断言不应同时为真的关系。推理引擎自动削弱支持较少的断言。
- **撤回（retraction）**：由原作者发起，将该知识节点关联的所有推理算子的 probability 设为 0。该知识节点变成孤岛，belief 自然回到 prior。撤回不是擦除历史，而是使某个断言失去证据支持。

具体的信念传播算法参见 [belief-propagation.md](belief-propagation.md)。

可废止性不是缺陷或特殊情况——它是区分科学推理与演绎证明的核心特征。在 Gaia 中，非单调性（新前提可以降低旧结论的信念）被内建于推理引擎中。

## 8. 审查/策展边界

本体论不将审查制品混入知识内容。

- 策展发现的潜在矛盾**尚不是**已接受图中的矛盾断言
- 佐证/独立证据发现**不是**推理算子
- infer 阶段的推理链接**尚不是**具有明确语义的 permanent 算子——但它仍然参与推理。infer 使用通用的默认参数，只是其具体的推理类型尚未经过审查确认。审查的作用是将 infer 细化为具有精确规则的 permanent 算子，而非决定它是否参与推理。

这些制品在被相应工作流接受之前，仍然是审查/策展制品。但"未审查"不等于"不参与推理"。

## 9. 对后续文档的约束

后续文档应遵循以下规则：

1. plausible-reasoning.md 应解释科学推理为何需要这一本体论
2. 编写语言规范应仅暴露面向作者的子集
3. Graph IR 应定义已接受本体论对象的结构表示
4. 推理理论应仅在承载 BP 的子集上定义 BP 算子合约
5. 审查/策展文档应定义非 BP 制品如何被提议、审计和可能被接受
6. 推理算子的生命周期（infer → candidate → permanent）应在审查流程文档中详细描述
